The wrapper code used to extract twitter data using the API tool could be found here.

The process of collecting data began with a review of the literature and examination of various sources, including the developer API and Kaggle. However, since the scope was limited to the University of Colorado Boulder, specific Twitter metadata was required. To address this, the snscrape tool was utilized. SNScrape API accesses the archived Twitter repository through asynchronous ten-fold developer accounts obtained over time. It provides a powerful search query that can be passed to the endpoint call, but the user does not need any API key. The returned JSON object contains several meta-data information that could be mined.

For the purposes of the project, a wrapper was created to make data retrieval easier, returning only the necessary attributes in a list form. The request body that could be used for the original endpoint looked like:

"{search_query} since:{YYYY-MM-DD} until:{YYYY-MM-DD}"

Example of a sample POST request made via the original tool:

										import os
				import pandas as pd
				#!pip install git+https://github.com/JustAnotherArchivist/snscrape.git

				search_query = "cuboulder"
				start_date = "2022-12-01"
				end_date = "2022-12-03"
				message_body = '{} since:{} until:{}'.format(search_query, start_date, end_date)
				print(message_body)
				# returns a generator with json response
				historical_metadata_raw = sntwitter.TwitterSearchScraper(message_body).get_items()
				historical_metadata = [tweet for tweet in historical_metadata_raw]

OVERVIEW

Principal components analysis (PCA) refers to the process by which principal components are computed, and the subsequent use of these components in understanding the data. 

PCA is an unsupervised approach, since it involves only a set of features 𝑋1,𝑋2,...,𝑋𝑝, and no associated response 𝑌.

When there are large number of features which may be correlated with each other (pair-plot), principal components analysis (PCA) allows us to summarize this set with a smaller number of representative variables that collectively explain most of the variability in the original set.

Apart from producing derived variables for use in supervised learning problems, PCA also serves as a tool for data visualization (visualization of the observations or visualization of the variables). It can also be used as a tool for data imputation — that is, for filling in missing values in a data matrix.

What is Clustering ?

Clustering is an unsupervised machine learning technique, which partitions the entire dataset into a fixed number of clusters or groups in such a way that observations in each group exhibit similar behavior or characteristics and are homogeneous. The observations in the same group are more similar than the observations in other groups.

Clustering Algorithms:

Based on the type of clustering algorithm used, there are different types of clustering methods:

- Hierarchical clustering: It is a tree based clustering method where the observations are divided into a tree like structure using distance as a measure.

- Centroid based clustering: In this method, the observations are partitioned into a predefined number of clusters (k) such that within cluster variances is minimum. k-means clustering is the most commonly used centroid based clustering algorithm.

- Density-based clustering: In this method, the observations are grouped together based on density of points that are closely packed. DBSCAN and OPTICS are two popular density-based algorithms.

- Distribution-based clustering: In this method, the observations are grouped based on the distribution models and observations in the same cluster are most likely coming from the same distribution. Gaussian mixture algorithm is an example of Distribution-based clustering.

How similarity is measured?

Similarity is computed using some form of distance matrix, in linear or non-linear space. Some distance metrics which are commonly used are:

Minkowski distance  calculates the distance between two real-valued vectors. It is a generalization of the Euclidean and Manhattan distance measures and adds a parameter, called the “order” or “p“, that allows different distance measures to be calculated.

Cosine similarity is used to determine the similarity between documents or vectors. Mathematically, it measures the cosine of the angle between two vectors projected in a multi-dimensional space. Cosine similarity says that to find the similarity between two points or vectors we need to find Angle between them.

Levenshtein distance, like Hamming distance, is the smallest number of edit operations required to transform one string into the other. Unlike Hamming distance, the set of edit operations also includes insertions and deletions, thus allowing us to compare strings of different lengths. 

Given two strings, the Levenshtein distance between them is the minimum number of single-character edits (insertions, deletions, or substitutions) required to change one string into the other.

Context

-  In our dataset (tweets) each sentence have been vectorized into a vector of [1 x 768] dimensions.

-  When we say that a text sentence is embedded into $768 \space dimensions$ by a pre-trained Transformer model, we mean that the sentence is **now represented as a vector of 768 numbers**, where each number in the vector corresponds to a different dimension in the embedding space.

-  These dimensions were learned by the Transformer model during training and represent different aspects of the meaning and context of the sentence.

-  For example, one dimension of the embedding vector might correspond to the presence of a certain topic or concept in the sentence, while another dimension might correspond to the sentiment or emotional tone of the sentence. By combining these different dimensions in a high-dimensional space, the Transformer model is able to capture the complex and nuanced meaning of a sentence.

OVERVIEW

Association rule mining is a technique used to identify patterns in large data sets. It involves finding relationships between variables in the data and using those relationships to make predictions or decisions. The goal of association rule mining is to uncover rules that describe the relationships between different items in the data set.

Apriori Algorithm

The algorithm was introduced by Mr. R. Agrawal and Mr. R. Srikant in 1994, they called it ‘Apriori’ because it uses prior information, i.e., existing transactions, to find out associations and patterns.

The algorithm aims to prune all the unwanted associations by first identifying the frequent item-sets in the dataset and then uses these frequent item-sets to generate association rules.

OVERVIEW

A decision tree is a type of predictive model that maps out possible outcomes and their likelihood based on a series of decision rules. It is a tree-like structure where each internal node represents a decision based on a feature or attribute, and each leaf node represents a class label or a predicted outcome. 

Based on Hunt’s algorithm, it could be used for Classification and Regression purposes (CART). The Decision Tree Hunt algorithm is a widely used approach for constructing decision trees. It is a recursive algorithm that starts with the entire dataset and selects the best feature to split the data based on a criterion such as information gain, GINI index, or entropy. 

The process of constructing a decision tree involves selecting the best feature or attribute to split the data at each node. There are several criteria for measuring the "goodness" of a split, such as:

- Gini gain (for categorical response)
- Information gain (for categorical response)
- Chi-Squared Test (for categorical response)
- Reduction in Variance (for continuous response)

Gini Gain
The Gini-gain measures the degree of impurity or randomness in a set of examples. A split is considered "good" if it leads to a significant decrease in the GINI impurity. Gini gain is measured by comparing gini impurity of parents and child nodes.

Information Gain
Entropy is another measure of impurity that quantifies the uncertainty in a set of examples. A split is considered "good" if it leads to a significant decrease in the entropy. Information gain is the measure of the expected reduction in entropy achieved by a split.

Chi-Squared Test
Chi-square is another method of splitting nodes in a decision tree for datasets having categorical target values. It is used to make two or more splits in a node. It works on the statistical significance of differences between the parent node and child nodes.

Reduction in Variance
It is a method for splitting the node used when the target variable is continuous, i.e., regression problems. It is called so because it uses variance as a measure for deciding the feature on which a node is split into child nodes. Variance is used for calculating the homogeneity of a node. If a node is entirely homogeneous, then the variance is zero.

It is generally possible to create an infinite number of decision trees because there can be multiple ways to split the data at each node, and the order in which the features are selected, eventually leading each leaf node to be a pure node. Additionally, decision trees can have multiple levels of depth, and the number of possible trees grows exponentially with the number of features and examples in the dataset.

Train-Test Split

Train-test split is a common technique in machine learning that involves dividing the dataset into training and testing sets. The training set is used to train the model, while the testing set is used to evaluate its performance on unseen data. Using scikit-learn's function train_test_split, with a test size of 30% and shuffled dataset, train and test sets were created.

Training Set




Testing Set









It is important to ensure that the training set and testing set are disjoint, meaning that there is no overlap between the two sets, because if there is overlap, the model can learn from the testing set and perform well on it, but it may not generalize well to new and unseen data. 

If the model has already seen the testing data during training, its performance on the testing set will not be a reliable indicator of its generalization performance.

OVERVIEW

Naive Bayes is a probabilistic algorithm that is commonly used for classification problems. It is based on Bayes' theorem, which provides a way to calculate the probability of a hypothesis given some evidence. 

In Naive Bayes, a naive  assumption is assumed that the features are independent given the class label, or the continuous features follow a normal distribution.

Despite this simplification, Naive Bayes can be very effective in many real-world classification tasks, especially when the number of features is large and the training data is limited. Naive Bayes is commonly used in natural language processing tasks, such as spam filtering and sentiment analysis, as well as in other applications such as image recognition and medical diagnosis.

There are three main types of Naive Bayes classifiers:

- Gaussian (for continuous  features)
- Multinomial (for categorical and longer text)
- Bernoulli (for shorter texts )

Gaussian Naive Bayes
This version assumes that the features are continuous and normally distributed. It models the distribution of each feature for each class using a Gaussian (normal) distribution.

Multinomial Naive Bayes
This version assumes that the features are discrete and counts or frequencies. It is commonly used for text classification (this project), where each feature corresponds to the frequency of a word in a document.

Bernoulli Naive Bayes
This version is similar to Multinomial Naive Bayes, but it assumes that the features are binary, meaning that each feature is either present or absent in the document. It is commonly used for binary classification problems, such as sentiment analysis or spam filtering.

Multinomial NB is generally used for longer documents/corpus and considers the word frequencies, whereas, Bernoulli NB is mostly used for shorter documents/corpuses and ignores the word count.

In Bernoulli Naive Bayes, the features are binary variables, which means that they can take on one of two values: 0 or 1. The algorithm calculates the probability of each feature being present in a class, and then uses Bayes' theorem to calculate the probability of the data belonging to each class. 

The Bernoulli Naive Bayes algorithm is commonly used for text classification, where the features represent the presence or absence of certain words in a document. It is also used in other applications where the features are binary, such as image classification or fraud detection.

OVERVIEW

Support Vector Machines (SVMs) are supervised learning algorithms that are primarily used for industry wise classification and regression applications. The goal of SVMs is to find the best way to draw a line that separates groups of things as far apart as possible. This line is determined by picking some points that are closest to it, called support vectors, and imagining a decision boundary to separate them in appropriate dimensions.

SVMs are linear separators because they assume that the classes can be separated by a hyperplane. However, when the classes are not separable in the original input space, SVMs utilize kernel functions to map the data points into a higher-dimensional feature space where they can be linearly separated. The choice of kernel function is important as it determines the shape of the decision boundary in the higher-dimensional space, and can be tuned by the Data Scientist.

The kernel is a function that takes two data points as input and produces a similarity score. Dot product is critical to a kernel function because it facilitates the computation of similarirty between two vectors. This is known as the "kernel trick", and it allows SVMs to efficiently compute the decision boundary in the transformed space. The dot product is a natural choice for measuring similarity because it captures the degree of alignment between two vectors.

SVMs Kernels
The choice of kernel function is important as it determines the shape of the decision boundary in the higher-dimensional space. Two common types of kernel functions are the polynomial kernel and the radial basis function (RBF) kernel.

Polynomial Kernel
The polynomial kernel maps the data points into a higher-dimensional space using a polynomial function of degree d, where d is a parameter that controls the degree of nonlinearity. Here's an example, let there be two points A and B in euclidean space, where A = (1, 2) and B = (2, 1), and a polynomial kernel with r=1 and d=2.

K(A,B) --> (ab + r)^d = (AB + 1)^2 = ((a1, a2)(b1, b2) + 1)^2= (a1b1 + a2b2 + 1)^2 = (a1^2b1^2 + 2a1a2b1b2 + a2^2b2^2 + 2a1b1 + 2a2b2 + 1)

A(a1,a2)-> {a1^2, sqrt(2)a1a2, a2^2, sqrt(2)a1, sqrt(2)a2, 1}
A(1, 2) --> {1,  sqrt(2)2,  4,  sqrt(2),  sqrt(2)2,  1}

B(b1,b2) > {b1^2, sqrt(2)b1b2, b2^2, sqrt(2)b1, sqrt(2)b2, 1}
B(2, 1) --> {4,  sqrt(2)2,  1,  sqrt(2)2,  sqrt(2),  1}


Radial Basis Function
The RBF kernel maps the data points into an infinite-dimensional space using a Gaussian function with a parameter r that controls the smoothness of the function.

OVERVIEW

Linear regression is a statistical method used for predictive analysis in machine learning, in order to model the relationship between a dependent variable and one or more independent variables. It is one of the simplest and intuitive regression techniques which has wide applications in data science and statistical analysis. It comes from a family of Generalized Linear Modelling where the response and the residual errors are assumed to have a Gaussian Distribution.

Primarily Linear regression are of two types: (1) Simple Linear Regression and (2) Multiple Linear Regression. In simple linear regression, we try to find the best fit line that can describe the relationship between a dependent variable (aka Response or Target) and one independent variable (aka Predictor or Feature). The equation of the line can be expressed as:

Response (Y) = Intercept + Slope x Feature (X)

The goal of Simple Linear Regression is to find the values of Intercept and Slope terms that minimize the difference between the predicted values and the actual values of the dependent variable. This is achieved by using a cost function such as the mean squared error (MSE), which measures the difference between the predicted values and the actual values, and tries to optimize the cost function on this difference.

Cost function
Cost function is a function that measures how well the model is fitting the training data. The goal of training a linear regression model is to minimize the value of the cost function. The most common cost function used in linear regression is the mean squared error (MSE), which is calculated as the average of the squared difference between the predicted values and the actual values of the dependent variable.

Optimization Algorithms
To optimize the cost function and find the best values of m and b that minimize the MSE, there are several optimization algorithms:

  • Ordinary Least Squares
  • Maximum Likelihood Estimation
  • Generalized Least Squares
  • Weighted Least Squares
  • Gradient Descent (Iterative approach)

Limitations of Linear Regression
  • There is an assumption of a linear relationship between the dependent variable and independent variable, which may not be always true for real-world use cases.
  • It is sensitive to outliers, which can affect the performance of the model.
  • There is an assumption of normally distributed residual terms or errors and also assumes they have a constant variance, which may not always hold true, especially, if the response is not continuous.
  • It can be influenced by the choice of independent variables, which can affect the accuracy of the model.

OVERVIEW

Neural networks are a type of machine learning model that is inspired by the structure and function of the human brain. They consist of interconnected neurons that process and transmit information, with each neuron receiving input from previous layers and producing output for subsequent layers. The final layer's output is the model's predicted output based on the input it received. Neural networks are commonly used in applications such as image and speech recognition and are particularly useful for complex pattern recognition and relationship detection in large datasets. The neural network is trained by adjusting the connection weights between neurons using backpropagation to minimize the difference between predicted and actual outputs, allowing for high accuracy and generalization.

Neural networks come in different types, including feedforward, recurrent, and convolutional neural networks. Feedforward neural networks are the simplest type, where data is processed in a single direction from input to output. Recurrent neural networks, on the other hand, allow for the processing of sequential data by incorporating feedback loops that enable previous outputs to be fed back as inputs. Convolutional neural networks are specialized for processing image data and utilize convolutional layers to detect features such as edges and shapes. With ongoing advances in machine learning and hardware, neural networks are becoming increasingly sophisticated and capable of tackling even more complex problems.

Types of Neural Networks
Neural networks can be grouped into different categories based on their structure and function. The feedforward neural network is the most basic type, with multiple layers of interconnected neurons that process data from input to output in a single direction. Each neuron in a feedforward neural network is linked to every neuron in the subsequent layer, with the output of each neuron in one layer serving as input for the next. Feedforward neural networks are widely used for tasks such as image classification and speech recognition, and their straightforward architecture makes them easy to train.

Another type of neural network is the recurrent neural network (RNN), which is tailored for sequential data processing, such as time series and natural language. RNNs differ from feedforward neural networks in that they have feedback loops that enable previous outputs to be used as inputs. This feature permits RNNs to capture temporal information and model complex input-output relationships over time. Nonetheless, RNNs can be challenging to train, and they suffer from the vanishing gradient issue that limits their ability to model long-term dependencies. To tackle this problem, specialized types of recurrent networks, such as Long Short-Term Memory (LSTM) and Gated Recurrent Units (GRU), have been developed that employ dedicated neurons to handle long-term dependencies better.
